# clickshare-signageV2

GitHub Pages-hosted digital signage system for Barco ClickShare Base Units. Each Base Unit is pointed at a URL that serves an image slideshow. Marketing manages content by uploading images to GitHub — no IT involvement needed for content updates.

---

## How it works

1. An image is added to `units/{facility}/{unit}/wallpapers/`
2. The push triggers a GitHub Actions workflow
3. The workflow runs `scripts/generate_manifests.py`, which scans each unit's wallpapers folder and writes an `images.json` manifest
4. Actions commits the updated manifests back to `main` with `[skip ci]`
5. GitHub Pages serves the updated content
6. Each Base Unit's slideshow polls `images.json` every 2 minutes and reloads if the content changed

End-to-end update time: ~4 minutes worst case (2 min pipeline + 2 min poll).

## Repository structure

```
clickshare-signageV2/
├── .github/
│   └── workflows/
│       └── update-manifests.yaml   ← triggers on wallpaper changes
├── scripts/
│   └── generate_manifests.py       ← scans wallpapers, writes images.json per unit
├── units/
│   └── {facility}/
│       └── {unit}/
│           ├── index.html          ← slideshow player (identical across all units)
│           ├── images.json         ← auto-generated, do not edit manually
│           └── wallpapers/
│               └── .gitkeep
├── .nojekyll                       ← disables Jekyll, enables direct file serving
└── README.md
```

Unit folders follow a fixed two-level structure: `units/{facility}/{unit}/`. The `index.html` uses a relative fetch path for `images.json`, so each unit automatically loads its own manifest.

## Adding a new unit

1. Create the folder: `units/{facility}/{unit}/wallpapers/`
2. Add a `.gitkeep` inside `wallpapers/` so git tracks the empty folder
3. Push — the workflow will generate `images.json` on the next image upload
4. Configure the Base Unit via the REST API (see ClickShare configuration below)

---

## Supported image formats

`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`

Slides are ordered alphabetically by filename. Naming images with a numeric prefix (`01-lobby.jpg`, `02-cafe.jpg`) controls display order.

---

## images.json

Auto-generated by the workflow. Do not edit the `images` array manually — it will be overwritten on the next push. The `interval` and `transition` fields are safe to edit and will be preserved across regenerations.

```json
{
  "images": ["slide1.jpg", "slide2.jpg"],
  "interval": 15000,
  "transition": 2000
}
```

| Field | Default | Description |
|---|---|---|
| `interval` | `15000` | Time per slide in milliseconds |
| `transition` | `2000` | Crossfade duration in milliseconds |

## GitHub Actions workflow

**File:** `.github/workflows/update-manifests.yaml`

**Trigger:** Push to `main` that modifies any file under `units/*/*/wallpapers/`

**Steps:**
1. Checkout the repo
2. Run `scripts/generate_manifests.py`
3. Stage `units/*/*/images.json`, commit with `[skip ci]`, and push — skipped if nothing changed

The `[skip ci]` tag prevents the workflow from re-triggering on its own commit.

---

## generate_manifests.py

**File:** `scripts/generate_manifests.py`

Walks `units/{facility}/{unit}/` looking for a `wallpapers/` subdirectory. For each unit found:
- Scans `wallpapers/` for supported image extensions
- Loads existing `images.json` to preserve `interval` and `transition` if already set
- Writes a new `images.json` sorted alphabetically

Can be run locally for testing:

```bash
python3 scripts/generate_manifests.py
# or with a custom path:
python3 scripts/generate_manifests.py --units-dir /path/to/units
```

## ClickShare configuration

Each Base Unit is configured via its REST API:

```
PATCH https://{base-unit-ip}:4003/v2/configuration/features/digital-signage
{
  "enabled": true,
  "url": "https://azleblanc.github.io/clickshare-signageV2/units/{facility}/{unit}/"
}
```

Authentication: HTTP Basic. Base Units use self-signed certificates — pass `--insecure` (curl) or `-SkipCertificateCheck` (PowerShell) for API calls on the internal network.

## Future plans

### Management landing page
A web UI where marketing can view all units, see loaded images, and adjust `interval`/`transition` per unit without touching JSON directly. Requires a backend (e.g. a Cloudflare Worker) to authenticate with the GitHub API and commit uploads — GitHub Pages alone cannot do this. The `images.json` contract is already the right interface for this UI to read and write.

### Content scheduling
Images with `startDate`/`endDate` metadata, filtered client-side by the slideshow JS.

### Image optimization
A GitHub Action to auto-compress and resize uploads via ImageMagick before committing.

### Content approval workflow
Draft branch + PR review before content goes live.
